Report Data Descriptions

The following information is included in the verification reports available through the API and reporting UI.

Field Name Description
Request ID A unique identifier for the request
Request Timestamp The date and time of the original request or its most recent revision
Account ID An identifier for the account that submitted the request
Person Id A unique string which represents an individual's identifying data, can be used to recognize duplicate submissions with the same data
User ID An identifier for the user that submitted the request
Username The username of the user that submitted the request
Result
  • true if the individual has been matched in at least one Verification Source and currently has at least one of the requested affiliation types
  • false if the individual has been matched in at least one Verification Source and does not currently have at least one of the requested affiliation types
  • [empty] if the individual has not been matched in any Verification Source
Status
  • NEW if the request has not been submitted to any Verification Source
  • PENDING if the request is still awaiting a final response from one or more Verification Source
  • OPEN if the request has been submitted to one or more Verification Sources but the individual has not been matched
  • COMPLETE if the request has been submitted to one or more Verification Sources and the individual has been matched in at least one
Errors A list of errors encountered during the verification process for this request
OrganizationEntity ID The identifier for the requested organization
Organization Name The name of the requested organization
Organization State The State (location) of the requested organization
Organization Country The Country (location) of the requested organization
OrganizationEntity Type The type of organization represented by the request
Verification Types The type(s) of verification used to obtain a verification result
Requested Affiliation Types The types of affiliations that the request was attempting to verify
Confirmed Affiliation Types The types of affiliations that the individual was determined to have
Data Updated Date The date the affiliation information was most recently updated by the data source
Period Begin Date The start of the most recent affiliation status period
Period End Date The end of the most recent affiliation status period
various Zero or more fields which represent the information of the individual being verified. The presence and number of fields will vary depending on the requested report and the target account's data retention policies.
various Zero or more fields which represent additional information collected with the verification request. The presence and number of fields will vary depending on the requested report and the target account's configuration.
Response Data
Error Codes
